# docs

## Purpose

Central documentation for Nexus product direction, architecture, compliance planning, and development practices.

## Current Status

Active and evolving. Serves as the source of truth during the architecture reset.

## What Belongs Here

- product docs in [`product/`](product/)
- architecture docs in [`architecture/`](architecture/)
- ADRs in [`decisions/`](decisions/)
- compliance planning notes in [`compliance/`](compliance/)
- setup and engineering guidance in [`development/`](development/)

## What Does Not Belong Here

- marketing claims about production readiness
- compliance certification claims without implementation evidence
- stale documentation that references removed repo surfaces

## Notes

Keep updates concise and factual. Mark incomplete areas as current/planned/parked so future contributors can reason about implementation status quickly.
