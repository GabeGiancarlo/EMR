# core

## Purpose

Domain-layer seed crate for reusable business concepts and contracts.

## Current Status

Planned/foundation. Contains early domain abstractions, not a complete product model.

## What Belongs Here

- domain entities and value objects
- domain service/repository traits
- shared domain errors and invariants

## What Does Not Belong Here

- HTTP framework wiring
- SQL query implementation details
- frontend-specific DTO concerns

## Notes

Recovery-specific domain modeling still needs expansion and validation against Clinician/Billing/Executive workflows.
