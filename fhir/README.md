# fhir

## Purpose

FHIR interoperability layer for future integrations.

## Current Status

Parked/future area. Not a primary active workflow path today.

## What Belongs Here

- FHIR client abstractions
- mapping/conversion utilities
- import/export integration logic

## What Does Not Belong Here

- core Nexus workflow logic for Clinician/Billing/Executive paths
- direct ownership of non-interoperability domain rules

## Notes

FHIR is important for interoperability, but current product priority is role-based workflow foundation in the backend.
